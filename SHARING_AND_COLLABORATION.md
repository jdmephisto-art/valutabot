# Как поделиться кодом с другим программистом

Вы задали отличный вопрос! Совместная работа — ключ к решению сложных проблем. Поскольку я, как AI-ассистент, не могу настроить для вас систему контроля версий, вот пошаговая инструкция, как вы можете поделиться кодом с другим разработчиком, используя стандартные инструменты — Git и GitHub.

## Шаг 1: Скачайте код проекта в виде ZIP-архива

Вы правы, этот шаг самый важный! Давайте разберем его подробнее. Так как я — AI-ассистент, я не вижу интерфейс так, как видите его вы, но в большинстве подобных онлайн-редакторов кода, как Firebase Studio, эта функция находится в одном из следующих мест:

*   **Верхнее меню "Файл" (File):** Посмотрите в самый верхний левый угол экрана. Там может быть меню с названием "Файл". Нажмите на него и поищите пункт, который называется "Скачать ZIP" (Download ZIP), "Экспорт" (Export), или "Скачать проект" (Download Project).
*   **Панель инструментов:** Осмотрите верхнюю часть интерфейса, где расположены различные иконки (сохранить, открыть и т.д.). Среди них может быть иконка, изображающая стрелку вниз, облако со стрелкой или архив (папку с молнией). Наведите на неё курсор — скорее всего, появится подсказка "Скачать код" или что-то похожее.
*   **Меню проекта:** Иногда рядом с названием вашего проекта есть иконка с тремя точками (...) или шестеренкой (⚙️). Нажмите на неё. В выпадающем меню может быть опция для скачивания.

**Что делать после нажатия:**

1.  После того, как вы нажмете на эту кнопку или пункт меню, браузер начнет скачивание ZIP-архива. Он будет назван, скорее всего, как ваш проект (например, `currency-bot-app.zip`).
2.  Найдите этот файл в папке "Загрузки" (Downloads) на вашем компьютере.
3.  Распакуйте архив в удобную для вас папку. Например, создайте на рабочем столе папку `MyProjects` и извлеките содержимое архива туда.

Теперь у вас на компьютере есть полная копия проекта, и вы готовы к следующему шагу — созданию Git-репозитория.

## Шаг 2: Установите Git (если его нет)

Если на вашем компьютере не установлен Git, скачайте и установите его с официального сайта: [git-scm.com](https://git-scm.com/).

## Шаг 3: Создайте локальный Git-репозиторий

1.  Откройте терминал или командную строку.
2.  Перейдите в папку с вашим проектом, которую вы распаковали на шаге 1.
    ```bash
    cd путь/к/вашему/проекту
    ```
3.  Выполните следующие команды, чтобы создать локальный репозиторий и добавить в него все файлы проекта:
    ```bash
    git init
    git add .
    git commit -m "Initial commit"
    ```

## Шаг 4: Создайте репозиторий на GitHub

1.  Зайдите в свой аккаунт на [GitHub.com](https://github.com/) (или создайте новый).
2.  Нажмите на кнопку "New" (Новый) для создания нового репозитория.
3.  Дайте ему имя (например, `currency-bot-app`), выберите, будет он публичным или приватным, и **не ставьте** галочки "Add a README file", "Add .gitignore" или "Choose a license", так как все эти файлы у вас уже есть.
4.  Нажмите "Create repository".

## Шаг 5: Свяжите локальный репозиторий с GitHub и загрузите код

GitHub покажет вам инструкции. Вам понадобятся команды из раздела "...or push an existing repository from the command line":

1.  Скопируйте и выполните первую команду, чтобы связать ваш локальный репозиторий с удаленным на GitHub. Она будет выглядеть примерно так:
    ```bash
    git remote add origin https://github.com/ваш-юзернейм/currency-bot-app.git
    ```
2.  Выполните следующие две команды, чтобы загрузить ваш код на GitHub:
    ```bash
    git branch -M main
    git push -u origin main
    ```

## Шаг 6: Пригласите другого программиста

1.  В вашем репозитории на GitHub перейдите во вкладку "Settings" (Настройки).
2.  В боковом меню выберите "Collaborators" (Сотрудники).
3.  Нажмите "Add people" (Добавить людей) и введите юзернейм или email программиста, которого хотите пригласить.

Готово! Теперь другой разработчик получит доступ к коду, сможет его скачать, вносить изменения и предлагать решения. Вы сможете вместе работать над проектом, используя всю мощь системы контроля версий Git.
